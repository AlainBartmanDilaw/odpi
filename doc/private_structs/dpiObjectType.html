ODPI-C Private Structure dpiObjectType¶

This private structure represents types such as those created by the SQL command CREATE OR REPLACE TYPE and is available by handle to a calling application or driver. The implementation for this type is found in dpiObjectType.c. Object types are created by calling the function dpiConn_getObjectType() or implicitly when calling the function dpiStmt_getQueryInfo() on a query column that contains objects. They are also created implicitly by calling the function dpiObjectAttr_getInfo() on an attribute that contains objects or by calling the function dpiObjectType_getInfo() on a collection that contains objects. Object types are destroyed when the last reference to them is released by the call dpiObjectType_release(). All of the attributes of the structure dpiBaseType are included in this structure in addition to the ones specific to this structure described below.

dpiConn *dpiObjectType.conn¶

Specifies a pointer to the dpiConn structure which was used to create this structure.

OCIType *dpiObjectType.tdo¶

Specifies the OCI TDO (type descriptor) handle.

OCITypeCode dpiObjectType.typeCode¶

Specifies the OCI type code.

const char *dpiObjectType.schema¶

Specifies the schema of the type, as a byte string in the encoding used for CHAR data.

uint32_t dpiObjectType.schemaLength¶

Specifies the length of the dpiObjectType.schema member, in bytes.

const char *dpiObjectType.name¶

Specifies the name of the type, as a byte string in the encoding used for CHAR data.

uint32_t dpiObjectType.nameLength¶

Specifies the length of the dpiObjectType.name member, in bytes.

dpiDataTypeInfo dpiObjectType.elementTypeInfo¶

Specifies the type of data of the elements of the collection. This is a structure of type dpiDataTypeInfo and is only valid if the object type refers to a collection.

boolean dpiObjectType.isCollection¶

Specifies if the type refers to a collection (1) or not (0).

uint16_t dpiObjectType.numAttributes¶

Specifies how many attributes the type has.
